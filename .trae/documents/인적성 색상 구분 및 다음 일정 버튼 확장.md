- 기존 데이터가 한글 라벨 등으로 저장돼 있어도 상태가 **정상 복원**되게 만들기.

## 구현 계획(코드 수정)
1) **상태 파싱 호환성 강화**
- [job_deadline.dart](file:///c:/Deadline%20Note/lib/src/models/job_deadline.dart#L86-L127)에서 `status` 파싱을 보강합니다.
  - 1차: 기존처럼 `JobStatus.name` 일치 시 그대로 사용
  - 2차: 과거/별칭 문자열 매핑(예: `"화상면접"`, `"인적성검사"` → `JobStatus.videoInterview`, `"서류"` → `document` 등)
  - 3차: `JobStatus.values` 중 `s.label == statusName` 매칭(라벨 기반 복원)
- 이렇게 복원된 `rawStatus`를 기존 규칙대로 `status`로 정규화합니다(현재 로직 유지).

2) **달력 월간 칩에 상태 뱃지(서/인/1차…)를 표시해 확실히 구분**
- [calendar_screen.dart](file:///c:/Deadline%20Note/lib/src/screens/calendar_screen.dart#L253-L277) 월간 날짜 셀에서 일정 칩 UI를 `Row`로 바꿔 작은 뱃지(서/인/1차/2차/최종)를 왼쪽에 추가합니다.
  - 배경색이 비슷해도 뱃지로 즉시 식별 가능
  - 기존 색 매핑은 유지하되, 필요하면 인적성 칩에 얇은 테두리(secondary)도 추가해 대비를 더 키웁니다.

3) **‘다음 일정’ 버튼을 인적성/면접 단계에서도 노출**
- [deadline_editor_form.dart](file:///c:/Deadline%20Note/lib/src/widgets/deadline_editor_form.dart#L472-L531)에서
  - `isDocumentStage = widget.initial.status == JobStatus.document`를 제거하고,
  - `canAddNext = _status.isPipelineStage && _outcome == JobOutcome.none` 같은 **현재 상태 기반 조건**으로 변경합니다.
  - UI는 “드롭다운(상태)”는 항상 두고, 옆(또는 아래)에 “다음 일정” 버튼을 추가하여 **상태 변경과 다음 일정 추가를 동시에 가능**하게 합니다.

4) **‘다음 일정 추가’ 시트의 전형 옵션을 “현재 단계 이후”로 제한**
- [deadline_editor_form.dart](file:///c:/Deadline%20Note/lib/src/widgets/deadline_editor_form.dart#L125-L131)에서
  - 현재는 `document`만 제외하고 전형을 보여서, 인적성에서도 다시 인적성을 고를 수 있습니다.
  - `_status`를 기준으로 `kPipelineStageOptions`에서 현재 인덱스 이후만 옵션으로 제공(예: 인적성 → 1차/2차/최종).
  - 다음 단계가 없으면 안내 스낵바(예: “다음 전형이 없어요.”) 후 종료.

5) (옵션) **‘합격’ 토글 시 다음 일정 자동 유도 범위를 확장**
- 현재는 서류에서만 합격 토글 시 다음 일정 시트를 자동으로 엽니다. [deadline_editor_form.dart](file:///c:/Deadline%20Note/lib/src/widgets/deadline_editor_form.dart#L540-L553)
- 이를 `canAddNext` 조건일 때로 확장해 인적성에서도 합격 토글 시 바로 다음 단계 계획을 잡을 수 있게 합니다.
